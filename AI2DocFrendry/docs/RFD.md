# RFD — AI2DocFrendry

> 版: 1.0 ／ 2026-07-21 ／ ma-garin
> **v0.2 から全面改訂。**[`RESEARCH.md`](RESEARCH.md) の調査結果により方針を変更。

---

## 1. 課題

社内ナレッジは多様な拡張子に分散しており、そのままRAGに投入すると精度が出ない。**調査の結果、原因の優先順位が確定した。**

| # | 問題 | 実証された効果量 |
|---|---|---|
| P-01 | **情報が抽出できていない／構造が壊れている** | パーサ選択だけで**22pt**、OCRエンジン選択で**20.2pt**、ingestion改善で**+15.2pt** |
| P-02 | **書式が意味を持つのに捨てられる** | 打ち消し線等でOCR精度82.9%に対しRAG精度**52.8%**（−30.1pt） |
| P-03 | **チャンクが構造を無視して切られる** | 表ヘッダ切断で**−27%**、階層分割の有無で表依存質問に**33pt**差 |
| P-04 | 文書が自己完結していない（「上記の通り」「別紙参照」） | 生成品質に効く（小型モデルで+0.078）。**文書をまたぐ参照の研究は存在しない** |
| P-05 | 表記ゆれ・書式の不統一 | Formatting Noise。**モデルが強いほど影響が減る＝逓減資産** |
| P-06 | 機密混入・陳腐化・重複 | 矛盾文書の存在で平均スコア−18.2% |

そして最大の問題は、**これらが投入後まで発覚しないこと**である。投入前にコーパス品質を測る枠組みは存在しない（評価フレームワーク10種すべてがクエリを必須とする）。

## 2. 提案

**AI2DocFrendry** — 社内ナレッジを投入すると、

1. **抽出・構造保持**（★最重要）— 多形式から、表・見出し階層・読み順を壊さずに取り出す
2. **再構成** — 見出し付与、指示語・参照の実体化、表記ゆれ統一、メタ情報補完
3. **改変検証** — 再構成が原文を歪めていないか機械検証し、違反した変更を取り消す
4. **品質診断** — 残る問題を根拠付きで報告
5. **RAG投入形式で出力** — 構造保持Markdown＋チャンク済みJSONL＋変更差分

### 2-1. v0.2からの重心移動

| | v0.2 | **v1.0** | 根拠 |
|---|---|---|---|
| 中核 | 中身の再構成 | **抽出・構造保持が第一、再構成は第二** | ingestion +15.2pt に対し他の介入は+0.6〜0.9pt |
| 診断の観点 | 独自の6観点 | **ICIS 2025 のDQ次元＋OHRBench の2分類** | 既存枠組みが実在。自作は車輪の再発明 |
| Markdown | 「AIが読みやすいから」採用 | **消去法で採用。優位性を主張しない** | 記法の優位を示す統制実験は存在しない |
| 表 | Markdown表で出力 | **結合セル・階層ヘッダがあればHTML併記** | Markdownにcolspan/rowspanが存在しない |

### 2-2. 診断の枠組み（既存研究から流用）

**軸1: ノイズ2分類**（OHRBench, ICCV 2025）

| 分類 | 内容 | 重要度 |
|---|---|---|
| **Semantic Noise** | 情報そのものの欠落・誤り（抽出漏れ、OCR誤認識、意味を持つ書式の消失） | **最優先。モデルが強くなっても解決しない** |
| Formatting Noise | 表記ゆれ、書式の不統一 | 副次。モデル進化とともに影響が減る |

**軸2: データ品質次元**（ICIS 2025 / Wang & Strong 1996 拡張）

| 次元 | 本システムで測るもの |
|---|---|
| `Intrinsic` | 抽出の正確性、矛盾の有無 |
| `Contextual` | 重複、陳腐化、網羅性 |
| `Representational` | 構造の機械可読性、表記の一貫性、略語定義 |
| `Accessibility` | 抽出可否、文書間参照の解決可否 |
| `Accountability` | 機密混入、所管の明示 |

### 2-3. スコープ外

| # | 作らないもの | 理由 |
|---|---|---|
| X-01 | ベクトルDB・検索・回答生成 | 既存領域 |
| X-02 | 抽出エンジンの自作 | **パーサ選択が22pt動かす最重要要素。自作すれば確実に既存OSSに劣る** |
| X-03 | ナレッジ管理UI | 別領域 |
| X-04 | 要約 | 情報が失われる（要約を生成に使うと正答率35.8→22.2に崩壊） |
| X-05 | 原文にない情報の追加 | 検証で機械的に禁止 |

## 3. 判断軸

1. **機密安全性** — 外部送信しない
2. **原文忠実性** — 再構成が事実を歪めない
3. **抽出品質** — 情報を落とさない（★調査で最重要と確定）
4. コスト0円
5. 再現性

## 4. 決定事項（ADR）

| ID | 決定 | 根拠 |
|---|---|---|
| ADR-01 | 全処理をローカル完結。外部送信しない | 社内ナレッジは機密を含む |
| ADR-02 | **抽出・構造保持に最優先で投資する** | ingestion +15.2pt / パーサ22pt。他の介入は1桁小さい |
| ADR-03 | **抽出は既存OSSに委譲し、プラグイン構造で差し替え可能にする** | 自作は既存に劣る。かつ文書タイプ別に最適パーサが異なる |
| ADR-04 | 中間形式はMarkdown。**ただし優位性は主張せず、結合セルを含む表はHTMLを併記** | 記法の優位に統制実験がない。Markdownはcolspan非対応 |
| ADR-05 | **semantic chunking を採用しない。文書構造（見出し・段落）で分割する** | 否定的証拠4件以上。精度差1.65ptに対し計算時間が最大54,000倍 |
| ADR-06 | **オーバーラップは文書種別で切り替える**（表を含む=25%、含まない=極小） | 表では0%→25%で+9.8pt、表なしでは20%超でprecision −40% |
| ADR-07 | **表は行単位チャンク＋各行にヘッダを反復する** | 3GPP 21,824表の統制実験で一貫して有効 |
| ADR-08 | 本文を再構成する。ただし原本保持・全変更を差分提示・情報追加を機械検証で禁止 | 事実の捏造・欠落を防ぐ |
| ADR-09 | 再構成は「層1ルール → 層2ローカルLLM → 層3検証」。層2は無効化可能 | LLMなしでも動作させ、出力を検証で縛る |
| ADR-10 | 検証を通らない変更は適用せず原文のまま残す | 直しすぎより残存を選ぶ |
| ADR-11 | **診断は ICIS 2025 のDQ次元と OHRBench の2分類を土台にする。独自観点を作らない** | 既存枠組みが実在 |
| ADR-12 | **CER/WER を品質指標に使わない** | OCR精度82.9%でRAG精度52.8%の実例。相関が弱い |

## 5. リスク

| # | リスク | 対応 |
|---|---|---|
| RSK-01 | LLMが事実を捏造する | 層3の改変検証で検出しロールバック（最重要対策） |
| RSK-02 | 抽出が構造を壊す | パーサを文書タイプ別に選べるようにする。抽出率を診断で可視化 |
| RSK-03 | **意味を持つ書式（打ち消し線等）の消失** | 検出して警告。**目視レビューでは発見できないため機械検出が必須** |
| RSK-04 | スキャンPDF・チャートが取得できない | チャートはOCR経路では救えない（GT比−72%）。検出・警告に留め、OCRはフェーズ2 |
| RSK-05 | 診断スコアの妥当性が未検証 | 実ファイルでの妥当性検証工程（W-18）を必須とする |
| RSK-06 | 外部PDFのテキストレイヤーが描画と乖離（split-view PDF） | 学術PDFで乖離率6.81%。抽出結果の差分監査をフェーズ2で検討 |

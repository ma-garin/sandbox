# requirements.md — PMOサービスメニューツール

## 1. 誰のための何の機能か
- **対象ユーザー**: 見込み顧客・社内メンバー・経営層
- **目的**: 品質PMO・AIサービスのサービスラインナップを一覧・詳細閲覧し、問い合わせへ誘導する

## 2. 受け入れ基準

### 機能要件
| # | 要件 | 優先度 |
|---|---|---|
| F-01 | 全サービスをトップページでカード一覧表示できる | Must |
| F-02 | サイドナビ2階層でサービスに絞り込みナビゲートできる | Must |
| F-03 | 各サービスの詳細（説明・特徴・支援フロー・タグ）を表示できる | Must |
| F-04 | 問い合わせボタンからモーダルフォームを開ける | Must |
| F-05 | サービス名でインクリメンタル検索できる | Should |
| F-06 | パンくずリストで現在位置を把握できる | Should |

### 非機能要件（nfr-standards準拠・業務PC向け）
| # | 要件 | 基準値 |
|---|---|---|
| N-01 | ページ初回読み込み | 3秒以内（4G想定） |
| N-02 | ファイルサイズ | 500KB以内 |
| N-03 | アクセシビリティ | WCAG 2.1 AA（コントラスト比4.5:1以上） |
| N-04 | フォント | Google Fonts CDN、font-display:swap必須 |
| N-05 | 外部依存 | Google Fontsのみ（有料CDN不使用） |

### 除外要件
- モバイル最適化（業務PC専用、NFR準拠）
- バックエンド・DB（単一HTML完結）
- ログイン・認証機能

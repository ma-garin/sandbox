# 境界値分析（Boundary Value Analysis）

ISO/IEC/IEEE 29119-4 / ISTQB FL準拠。順序尺度を持つパーティションの境界とその隣接値を
テストする。同値分割で作ったパーティション表を前提とする（先に同値分割を実施すること）。

## 適用手順

1. **境界の特定** — パーティション表から、順序尺度を持つパーティション（数値範囲・文字列長・件数・日付等）の境界をすべて列挙する。境界1つにつき「パーティション内の端の値」と「隣接パーティション側の値」が存在する。
2. **値の選定方式を宣言** — 2値方式（境界上＋境界外）か3値方式（境界の前・上・後）かを選び、宣言に明記する。
   - 既定は**2値方式**。金額計算・安全関連など off-by-one の影響が大きい対象は3値方式。
3. **境界値一覧の作成** — 下記書式。各境界に根拠を必須で付ける。
4. **暗黙の境界の検討** — 仕様に明記されない技術的境界も検討し、採否を宣言する:
   - データ型の限界（0、-1、2^31-1、Number.MAX_SAFE_INTEGER）
   - 長さ0（空文字・空配列）と最大長
   - 日付の境界（月末・年末・うるう日・タイムゾーン跨ぎ）
   - これらは仕様に記述がないため `confidence: 推定` を付け、要人間確認に挙げる。

## 境界値一覧の書式

| # | パラメータ | 境界 | 方式 | テスト値 | 期待結果 | 根拠 |
|---|---|---|---|---|---|---|
| B1 | 注文数 | 下限=1 | 2値 | 0 / 1 | 拒否 / 受理 | spec.md「3.1」:「1〜100個」 |
| B2 | 注文数 | 上限=100 | 2値 | 100 / 101 | 受理 / 拒否 | 同上 |

## カバレッジ計算（宣言に使用）

```
境界値カバレッジ = テストケースが実行する境界値数 ÷ 識別した全境界値数 × 100%
```

（2値方式なら境界1つ＝境界値2個として数える。3値方式なら3個。）

## よくある我流化（禁止パターン）

- 境界「付近のキリのいい値」（0, 10, 100）だけ試して境界そのもの（1, 99, 101）を外す
- 上限だけテストして下限を忘れる
- 期待結果を「エラーになる」とだけ書き、どのエラー（メッセージ・ステータス）かを仕様から特定しない

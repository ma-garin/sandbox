# QA Knowledge Reverse Docs

## 概要

公開WebシステムのURLから画面要素を解析し、上流工程で使うドキュメントをルールベースで生成するPython製MVP。

初期版はAPI利用を前提にせず、HTML/DOMから見出し、リンク、入力項目、フォーム、外部参照を抽出してMarkdown/CSV文書を生成する。

## セットアップ

```bash
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
```

## 使い方

```bash
streamlit run app.py
```

ブラウザで表示されたURLにアクセスし、起点URLを入力して「解析して文書生成」を実行する。
同一ドメイン内リンク、`sitemap.xml`、`robots.txt` の Sitemap 指定を安全上限の範囲で自動探索し、取得できたページを統合して文書化する。
自動探索で拾えない画面は、追加URLとして手動補完できる。

## 生成ドキュメント

- 企画書
- システム概要書
- 要件定義書
- 業務フロー / ユースケース一覧
- アーキテクチャ設計書
- 画面一覧
- 画面仕様書
- 機能一覧
- 機能仕様書
- データ項目定義書
- 外部インターフェース仕様書
- 非機能要件定義書
- テスト観点表
- 狩野モデルUXレビュー
- RFD / ADR
- トレーサビリティマトリクス

各文書には、レビュー状態、根拠URL/根拠要素、確信度、未確認事項を含める。
レビュー状態は `未レビュー` / `要確認` / `確認済み` / `差戻し` を文書ごとに管理する。

## 表形式エクスポート

ZIP出力には、文書別Markdownに加えて次のCSVを含める。

- 画面一覧
- 機能一覧
- データ項目
- 外部IF候補
- 未確認事項一覧
- トレーサビリティ
- 引継ぎ阻害リスク
- 狩野モデルUXレビュー

## メモ

- MVP対象は公開HTMLページ。起点URLから同一ドメイン内リンクを自動探索する。
- ログイン後画面、JavaScriptレンダリング必須サイト、LLM補完は後続フェーズ。

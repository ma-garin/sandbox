## 画面構成

| 画面 | 役割 | 主なコンテンツ |
|------|------|----------------|
| トップ | 全体把握 | 章一覧、学習導線、検索、特集、更新情報 |
| 章トップ | 章の整理 | 章概要、学習順、関連法則、サブセクション |
| 法則詳細 | 知識の定着 | 定義、図解、適用例、NG例、QA観点 |
| レシピ詳細 | 実務活用 | 問題、原則、処方、評価方法、Before/After |
| 索引/検索 | 横断移動 | キーワード、タグ、章別、難易度別 |

## 共通レイアウト要素

- 左固定サイドメニュー（PC）/ 上部ドロワー（スマホ）
- パンくずナビ
- 章内目次（ページ内アンカー）
- 前後章ナビ
- 関連項目リンク
- タグ（「法則」「アクセシビリティ」「フォーム」などで横断検索）

## サイドメニュー要素

| 要素 | 役割 | 優先度 |
|------|------|--------|
| 章一覧 | 全体構造の把握と移動 | 高 |
| 章内目次 | 長いページ内の移動 | 高 |
| タグ | 横断検索 | 中 |
| 関連リンク | 近い概念・関連レシピへ | 中 |
| 進捗表示 | 学習済み/未学習の可視化 | 中 |

## GitHub Pages 実装方針

| 項目 | 方針 |
|------|------|
| 推奨構成 | VitePress / Astro / Docusaurus / plain HTML |
| サイズ上限 | GitHub Pages 1GB 以下 |
| 初回ロード | 軽量に保つ（遅延読み込み・章分割） |
| 図表 | SVG 中心 |
| 避けること | 巨大単一 HTML、無圧縮 PNG、大量 GIF、一括読み込み |

## コンテンツ規模（目標）

| コンテンツ種別 | 目安量 | 容量寄与 | 方針 |
|---------------|--------|----------|------|
| SVG図解 | 300〜500枚 | 中 | 自作で軽量にする |
| UIレシピカード | 200〜300本 | 中 | HTML/CSS 中心 |
| サンプル画面 | 100画面 | 高 | 実画面より模擬画面を採用 |
| 演習 | 100問以上 | 低 | 診断形式で追加しやすくする |
| 検索 index | 1〜5万項目 | 中 | 別ファイル化 |

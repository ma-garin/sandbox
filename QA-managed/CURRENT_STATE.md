- `_template/` を複製し `QA-managed/` を作成、README/AGENTS/CURRENT_STATE を初期化
- **MVP実装**（`app/`、ブラウザ完結・ゼロ依存）:
  - `viewpoints.js`: QA-PMOの63観点・12カテゴリ・12欠陥パターンを移植（根拠標準つき）
  - `analyzer.js`: 仕様テキスト→入力項目型/機能特性/業種をキーワードで決定的に推定（evidence保持）
  - `engine.js`: 観点適用→観点表・カバレッジ・関連欠陥・CSV生成（QA-PMO engine.py 移植）
  - `index.html`+`app.js`: 貼付→自動判定チップ(ON/OFF修正可)→観点表→CSVのUI
- **拡張実装**（4機能を追加）:
  - `engine.coverageReport()`: **観点カテゴリ網羅率**（分母=全カテゴリ、AI特性ありなら20/なしは12）＋**盲点（未カバーカテゴリ）検出**
  - `scaffold.js`: 観点表 → **Playwright(TS)/pytest 雛形**を生成（実行レイヤへの橋渡し）
  - `llm.js`: **任意のAI補完**（Claude API `claude-opus-4-8`、ブラウザ直叩き＋localStorageキー）で観点を前提/手順/データ/期待結果へ具体化。未設定時は決定的観点表にフォールバック
  - UI: 網羅率KPI・盲点バナー・雛形DL・APIキー管理・AIテストケース表示（⑤）
- **検証**: node単体（EC=36観点、AIチャット=32観点/網羅率90%/盲点2）＋ Chromium E2E（描画・再生成・雛形DL・AI無キー時の安全動作、JSエラーなし）
  - 注: AI補完の実API呼び出しは claude-api スキル準拠で実装。実キーでの往復は未実施（無キー時のフォールバックは検証済み）

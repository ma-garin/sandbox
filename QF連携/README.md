# QF連携

## 概要

Veriserve社「QualityForward」のREST API（`https://cloud.veriserve.co.jp/api/v2`）は、レスポンス構造が複雑（`category1..25`/`content1..10`等の生フィールド名）で、1操作に複数エンドポイントの組み合わせが必要、一覧の検索・フィルタが無いなど使いにくい点が多い。
本プロジェクトはそれをブラウザ完結（フロントのみ、ビルドレス）で使いやすく包むツール。APIキーはこのブラウザのlocalStorageにのみ保存される。

対応範囲: プロジェクト閲覧 / テストスイート・バージョン・テストケースのCRUD / テストフェーズ→サイクル→結果登録ウィザード / 不具合件数スナップショット登録とグラフ（「課題(Issue)管理」に相当するAPIが無いための代替）。

## セットアップ

```bash
# ビルド不要（vanilla JS ES modules）。ローカル静的サーバで配信するだけでよい。
# file:// では ES module の import がブロックされるため、必ずHTTP経由で開くこと。
cd QF連携
python3 -m http.server 8080
# ブラウザで http://localhost:8080/ を開く
```

## 使い方

1. 「設定」タブでQualityForwardのプロジェクトAPIキーを登録する（設定画面から発行、複数プロジェクトを行き来する場合は複数プロファイルを追加して切替可能）
2. 「プロジェクト」タブで接続確認
3. 「テストスイート」タブでテストスイート→バージョン→テストケースを作成・閲覧
4. 「テスト実行」タブでテストフェーズ→割当→サイクルを選択・作成し、テスト結果を登録
5. 「不具合件数」タブでOPEN/CLOSE件数の日次登録とグラフ確認（履歴はこのブラウザのみ保持）

テストの実行: `node tests/<name>.test.mjs`（実APIへの通信は行わない。fetch注入によるオフライン検証のみ）

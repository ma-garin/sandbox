# spec.md — TraceGate MVP（INF-01）

版: v1.0（2026-07-05確定）／ 変更はSonnet実装セッションでは禁止（→ ROADMAP.md 運用ルール3）

## 目的

仕様ファイル内の要件ID（`REQ-NNN`）とテストコード内のアノテーション（`@covers REQ-NNN`）を
突合し、トレーサビリティマトリクスの生成と乖離率によるCIゲート判定を行うCLIツール。
MVPは規約ベースの決定的処理のみ（LLM・AST解析は使わない）。

## 用語

- **要件ID**: 正規表現 `REQ-\d{3}` に一致する文字列（例: `REQ-001`）
- **カバレッジ宣言**: `@covers` を含む行に書かれた要件IDの列挙
- **孤立要件（uncovered）**: どのテストからも@coversされていない要件
- **孤立カバレッジ（orphan）**: 定義が存在しない要件IDへの@covers
- **乖離率（divergence）**: 孤立要件数 ÷ 総要件数

## 機能要件（EARS）

### 要件抽出

- REQ-001: システムは、指定された仕様ファイル群の各行を走査し、`REQ-\d{3}` に一致する全ての要件IDを抽出しなければならない。
- REQ-002: システムは、同一要件IDが複数回出現した場合、ファイルパス昇順→行番号昇順で最初の出現を定義として採用し、以降の出現を無視しなければならない。
- REQ-003: システムは、要件の定義行から要件IDおよび前後の記号（`#` `-` `*` `:` `|` 空白）を除去した残りのテキストを、その要件のタイトルとして記録しなければならない。

### カバレッジ抽出

- REQ-004: システムは、指定されたテストファイル群のうち文字列 `@covers` を含む各行から、その行に出現する全ての要件IDをカバレッジ宣言として抽出しなければならない。
- REQ-005: システムは、各カバレッジ宣言についてファイルパスと行番号（1始まり）を記録しなければならない。
- REQ-006: システムは、`@covers` を含まない行に出現する要件IDを、カバレッジとして扱ってはならない。

### 突合

- REQ-007: システムは、全要件を「カバー済み」「孤立要件」のいずれかに分類しなければならない。
- REQ-008: システムは、定義が存在しない要件IDへのカバレッジ宣言を孤立カバレッジとして列挙しなければならない。
- REQ-009: システムは、乖離率を（孤立要件数 ÷ 総要件数）で算出しなければならない。

### レポート出力

- REQ-010: システムは、Markdown形式のトレーサビリティマトリクス（design.md「出力例」の書式）を標準出力に出力しなければならない。`--output <path>` 指定時はファイルにも書き出す。
- REQ-011: システムは、`--json <path>` が指定された場合、design.mdのJSONスキーマに従うレポートを指定パスに書き出さなければならない。
- REQ-012: マトリクスの末尾サマリには、総要件数・カバー済み数・孤立要件数・孤立カバレッジ数・乖離率（小数第1位までの%）を含めなければならない。

### ゲート判定（終了コード）

- REQ-013: システムは、乖離率が `--max-divergence`（既定値 0.0）を超過した場合、終了コード1で終了しなければならない。
- REQ-014: システムは、孤立カバレッジが1件以上存在する場合、終了コード1で終了しなければならない。ただし `--allow-orphans` 指定時は終了コードに影響させない。
- REQ-015: システムは、抽出された要件が0件の場合、標準エラーに設定誤りを示すメッセージを出力し、終了コード2で終了しなければならない。
- REQ-016: システムは、ゲート判定を全て通過した場合、終了コード0で終了しなければならない。

### CLI

- REQ-017: システムは、`--specs <glob> [...]`（既定値 `specs/**/*.md`）と `--tests <glob> [...]`（既定値 `tests/**/*.*`）でファイル群を指定できなければならない。
- REQ-018: システムは、テストファイルのうち拡張子が `.py` `.ts` `.tsx` `.js` `.jsx` `.md` 以外のファイルを走査対象から除外しなければならない。
- REQ-019: システムは、`--version` 指定時にバージョン文字列を出力して終了コード0で終了しなければならない。

## 非機能要件

- NFR-001: Python 3.10以上・標準ライブラリのみで動作する（テストはpytestを使用）
- NFR-002: 合計1,000ファイル・10万行の入力を10秒以内に処理する（ファイルは1回だけ読む）
- NFR-003: UTF-8以外でデコードできないファイルは警告を標準エラーに出してスキップし、異常終了しない

## MVP対象外（明示）

- LLMによる意味照合（IDが機械的に紐付かない箇所の照合）
- ASTによるコード−要件の対応付け（テストアノテーション以外のトレース）
- Excel出力・仕様鮮度アラート・GitHub Checks API連携（Actionsの終了コード運用で代替）

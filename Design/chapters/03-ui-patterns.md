# Part 3: UIパターン・デザインレシピ

## この章で学ぶこと

頻出する UI 設計上の問題を「レシピ」として体系化する。問題 → 原則 → 処方 → 検証の4ステップで整理し、実務で即使えるパターン集にする。

---

## 3-1. レシピの基本フォーマット

```
Problem      どんな状況でユーザーが迷うか
    ↓
Principle    どの法則・原則で説明できるか
    ↓
Prescription UIをどう変えるか（具体的に）
    ↓
Evaluation   改善を何で測るか（成功指標）
```

---

## 3-2. 認証系レシピ

### ログインフォーム

| ステップ | 内容 |
|---------|------|
| Problem | パスワードを思い出せない / 入力ミスのフィードバックが遅い |
| Principle | エラー予防（#5）/ システム状態の可視化（#1） |
| Prescription | パスワード表示トグル追加。リアルタイムバリデーション。ソーシャルログインを目立つ位置に |
| Evaluation | ログイン成功率。パスワードリセット率。フォームエラー発生数 |

### オンボーディング

| ステップ | 内容 |
|---------|------|
| Problem | 初回起動で何をすべきか分からない |
| Principle | Jakob の法則（既存パターン）/ 認識 vs 記憶（#6） |
| Prescription | 3ステップ以内のセットアップウィザード。スキップ可能に。プログレス表示 |
| Evaluation | オンボーディング完了率。Day-1 / Day-7 継続率 |

---

## 3-3. 入力系レシピ

### フォーム設計

| ステップ | 内容 |
|---------|------|
| Problem | 入力項目が多くて途中で離脱する |
| Principle | Hick の法則（選択肢を減らす）/ テスラーの法則（複雑さの配置） |
| Prescription | 必須項目のみ表示。ラベルは入力欄の上に配置。グループ化でステップ分割 |
| Evaluation | フォーム完了率。離脱ポイント分析 |

### バリデーション・エラー表示

| ステップ | 内容 |
|---------|------|
| Problem | 送信後にまとめてエラーが出て修正箇所が分かりにくい |
| Principle | エラー回復の支援（#9）/ システム状態の可視化（#1） |
| Prescription | インラインリアルタイムバリデーション。エラーは入力欄直下に表示。修正方法を具体的に書く |
| Evaluation | バリデーションエラー発生後の完了率 |

---

## 3-4. 探索系レシピ

### 検索

| ステップ | 内容 |
|---------|------|
| Problem | 検索結果が 0 件でも何もしてくれない |
| Principle | エラー回復の支援（#9）/ 空状態パターン |
| Prescription | 0件時は「代替候補」「スペルチェック」「人気ワード」を提示。検索履歴の活用 |
| Evaluation | 検索後のクリック率。0件率 |

### フィルタ・ソート

| ステップ | 内容 |
|---------|------|
| Problem | フィルタを適用してもどれが効いているか分からない |
| Principle | システム状態の可視化（#1）/ 認識 vs 記憶（#6） |
| Prescription | 適用中のフィルタをチップで表示。1クリックでリセット可能に |
| Evaluation | フィルタ使用率。フィルタ後の購入・選択完了率 |

---

## 3-5. 一覧・詳細系レシピ

### テーブル（一覧）

| ステップ | 内容 |
|---------|------|
| Problem | 行が多くて目的の情報を探せない |
| Principle | Fitts の法則（クリックしやすい）/ 認知負荷の低減 |
| Prescription | 行ホバーハイライト。固定ヘッダー。重要列を左側に配置。行クリックで詳細 |
| Evaluation | 目的情報到達時間。ページ滞在時間 |

---

## 3-6. 状態系レシピ

### 空状態（Empty State）

| ステップ | 内容 |
|---------|------|
| Problem | データがない時に何も表示されずユーザーが困惑する |
| Principle | エラー回復の支援（#9）/ ユーザー制御と自由（#3） |
| Prescription | イラスト + 説明テキスト + CTAボタン。「なぜ空なのか」「次に何をすべきか」を伝える |
| Evaluation | 空状態からのCTA クリック率 |

### ローディング

| ステップ | 内容 |
|---------|------|
| Problem | 読み込み中か止まっているか分からずユーザーが不安になる |
| Principle | システム状態の可視化（#1）/ Doherty の閾値 |
| Prescription | 400ms 以上かかる処理にはスケルトンUI or スピナー表示。長い処理は進捗% を表示 |
| Evaluation | ローディング中の離脱率 |

### エラー回復

| ステップ | 内容 |
|---------|------|
| Problem | エラーが出たが原因も対処法も分からない |
| Principle | エラー回復の支援（#9）/ 現実世界との一致（#2） |
| Prescription | 平易な言葉でエラー原因を説明。具体的な対処手順を提示。サポートへの導線 |
| Evaluation | エラー後の再試行成功率 |

---

## 3-7. AI 系レシピ

### AIチャット UI

| ステップ | 内容 |
|---------|------|
| Problem | AI の回答が長くて正確なのか不正確なのか分からない |
| Principle | システム状態の可視化（#1）/ ユーザー制御と自由（#3） |
| Prescription | 根拠・出典を明示。再生成ボタン。修正指示テキストエリア。コピー・削除 |
| Evaluation | 再生成率。コピー率。タスク完了率 |

## 収録予定レシピ一覧

| カテゴリ | レシピ | 優先度 |
|---------|--------|--------|
| 認証 | ログイン、パスワードリセット、2FA、オンボーディング | 高 |
| 入力 | フォーム設計、バリデーション、ファイルアップロード | 高 |
| 探索 | 検索、フィルタ、ソート、オートコンプリート | 高 |
| 一覧 | テーブル、カード一覧、無限スクロール、ページネーション | 高 |
| 詳細 | 詳細表示、タブ切り替え、モーダル、ドロワー | 中 |
| 管理 | ダッシュボード、通知、バッジ、設定 | 中 |
| 状態 | 空状態、ローディング、エラー回復、オフライン | 高 |
| AI | AIチャット、提案UI、Human-in-the-loop | 中 |

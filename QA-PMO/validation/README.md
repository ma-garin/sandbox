# バリデーション研究

テスト観点ライブラリ（VeriServe）の有効性を、  
**事前登録方式**（臨床試験型・コミットにより凍結）で検証した研究。

---

## 研究の目的

「ドメイン特有の観点ライブラリは、一般的なチェックリストやAIプロンプトよりも  
本番障害の捕捉率が高い」という仮説を、公開バグデータで定量的に検証する。

---

## 方法論

| 項目 | 内容 |
|---|---|
| 設計 | 事前登録方式（評価方法を先にコミットし、その後バグ収集） |
| 対象 | Saleor Commerce の本番クローズ障害（2022〜2024年） |
| サンプル数 | N=20（公開GitHubイシューから機械的に選択） |
| 比較手法A | ISTQB Foundation Level + EC向けチェックリスト |
| 比較手法B | GPT-4o（標準プロンプト、観点指定なし） |
| 提案手法 | 観点ライブラリ v2.0.0（commit db1c882 で凍結） |
| 採点原則 | 疑わしい場合は「捕捉なし」（自己採点不利方向） |

---

## 最終結果

| 手法 | 全バグ捕捉率（N=20） | ドメイン特有バグ捕捉率（N=16） |
|---|---|---|
| ISTQB/一般チェックリスト | 5%（1/20） | 0%（0/16） |
| GPT-4o（標準プロンプト） | 10%（2/20） | 0%（0/16） |
| **観点ライブラリ（VeriServe）** | **85%（17/20）** | **88%（14/16）** |

ドメイン特有バグ（CONC/STATE/DATA/SEC/I18N カテゴリ）において  
ベースライン2手法の捕捉率は 0% であり、観点ライブラリは 88% を達成した。

## 検証の完全性について

**正直開示（バイアス排除）:**
- B-04（DB集計バグ）: 実装詳細特有の障害として観点ライブラリを「捕捉なし」と判定
- B-18（Avatax再試行バグ）: 外部SaaS固有の設定問題のため「捕捉なし」と判定

採点の詳細は `SCORING.md` に記録されている。

## 解釈上の注意

- N=20 のため統計的検出力は限定的。大きな差（88% vs 0%）は示されているが、業種・プロジェクト規模による変動が予想される
- ROI計算機（platform/ 内）はこの研究結果を使って年間削減効果を試算できる
- 比較手法Bには「観点ライブラリを参照させたGPT-4o」は含まれていない（それは提案手法側になる）

# QA-managed — 観点駆動マネージドQaaS（国産QA Wolf）

## 概要

**「上流の観点設計をAIで握り、実行はAI＋人のマネージドで回す」** 国産のマネージド型
E2Eテストサービス（QaaS: QA as a Service）の設計・検証リポジトリ。

米 QA Wolf の「AI＋人でE2Eカバレッジを短期に引き上げ、成果ベースで課金する」モデルを
土台にしつつ、彼らが**人手に頼る上流工程（何をテストすべきかの観点設計）をAIで前倒し**する
ことで、QA Wolf の構造的弱点（テスト増加＝人件費増）を回避する。第三者検証会社の
**中立性・観点ナレッジ・エンジニア供給網**を差別化の源泉とする。

> 位置づけ: 実行フェーズの `QA-PMO`、上流レビューの `spec-inspector` / `QA-knowledge` の
> 知的資産（観点ライブラリ63観点・欠陥パターン等）を束ね、**サービスとして売る**ための器。

## なぜ作るか（市場仮説）

| 論点 | 内容 |
|---|---|
| 欧米 | Gartnerが2025/10に初のMQ公開、Forresterが「Autonomous Testing Platforms」へ改称。エージェント型・自己修復はレッドオーシャン化 |
| 国内 | 自動化(Autify)・VRT(NTTテクノクロス AIspector)は充足しつつある。一方 **QaaSマネージドの国産主体**と**テスト観点の自動設計**は空白（ブルーオーシャン） |
| QA Wolf の弱点 | 保守・トリアージが労働集約 → テスト増でコスト膨張。年間契約中央値 ≒ $90k と高額 |
| 当社の勝ち筋 | **①観点設計をAIで握る（上流）× ②実行はマネージド** で、QA Wolf が効率化できていない上流を差別化点にする |

## サービス構想（QA Wolf との対比）

| 工程 | QA Wolf | QA-managed（本構想） |
|---|---|---|
| ① 観点設計（何をテストするか） | 人がインタビュー＆クロールで手作業 | **観点ライブラリ×AIで自動設計**（差別化の核） |
| ② テスト生成 | マルチエージェントAI→Playwright/Appium | 同左（AI生成）＋人がレビュー承認 |
| ③ 実行 | 数千Dockerで100%並列（約3分） | 並列実行（MVPはローカル/CIスモーク） |
| ④ トリアージ | AI一次調査→人が確認、24h SLA | AI一次調査→**第三者検証者が中立判定** |
| 課金 | テスト単位の月額従量（$40〜/本） | 成果ベース（カバレッジ/観点網羅）を検討 |
| 保証 | ゼロフレーク／4ヶ月で80%カバレッジ | 観点網羅率＋evidence-only 判定を保証軸に |

- 既存資産の再利用が前提：`QA-PMO/portal/knowledge`（観点エンジン・63観点）、
  `spec-inspector`（観点スコアリング）、`QA-knowledge`（URL→上流ドキュメント逆生成）。

# AI2DocFrendry

## 概要

多様な拡張子で散在する**社内ナレッジを、生成AI（RAG）が読み込みやすい形式へ変換**し、
さらに**変換結果のAI可読性を6観点で採点して改善提案を返す**、ローカル完結型のRAG前処理支援システム。

```
[社内ナレッジ]              [AI2DocFrendry]                    [出力]
 docx/xlsx/pptx  ──┐                                    ┌── Markdown（構造化済み）
 pdf/txt/md/csv  ──┼─→ 取込 → 抽出 → 正規化 → 構造化 ─┼── JSONL（チャンク済み・RAG投入用）
 html/eml        ──┘         ↓                          ├── metadata.csv（出典・機密区分）
                          採点・診断  ────────────────→ └── 品質レポート（スコア＋指摘＋改善提案）
```

### 何が新しいか

変換ツール自体は既存OSSで足りている。本システムが埋めるのは、**「RAGに投入する前に、AIが読める状態かを判定する」**という空白領域。

| AI可読性6観点 | 判定するもの |
|---|---|
| 構造明示性 | 見出し・箇条書き・表がマークアップされているか |
| 自己完結性 | 「別紙参照」「上記の通り」など、チャンク単体で意味が通らない記述 |
| 表現一貫性 | 表記ゆれ・用語ゆれ・全半角混在 |
| 抽出性 | テキストとして取り出せた割合（スキャンPDF・画像内の表を検出） |
| 文脈情報量 | 版数・更新日・所管・略語定義の有無 |
| 安全性 | 機密情報の混入・陳腐化・重複 |

指摘は **ISTQB severity（Critical/High/Medium/Low）** で分類し、**根拠となる引用を必ず添える（evidence-only）**。
総合判定は A（そのまま投入可）／B（投入可）／C（要改善）／D（投入非推奨）の4段階。

## 設計方針

- **ローカル完結** — 社内ナレッジは機密を含むため、既定構成で外部通信を一切行わない（ADR-01）
- **コスト0円** — 無料OSSのみ。外部登録・従量課金なし
- **決定的** — 採点はルールベース。同一入力に対し常に同一出力（ADR-05）
- **本文を書き換えない** — 改善提案までに留める。社内文書の改変は責任所在が曖昧になるため（ADR-04）

## メモ

- 変換エンジンは自作せず既存OSS（個別パーサ）に委譲し、プラグイン構造で差し替え可能にする。
- スコープ外: ベクトルDB・検索・回答生成（LangChain/LlamaIndex等の領域）、ナレッジ管理UI。
- 本プロジェクトは学習・検証目的（sandbox配下）。

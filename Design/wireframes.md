# ワイヤーフレーム定義

4画面の骨格を定義する。まずこの4画面を固めることで、後続の大量コンテンツ投入でも構造が崩れにくくなる。

---

## ワイヤー 1: トップページ

```
┌─────────────────┬─────────────────────────────────────────────────────────┐
│                 │                                                         │
│   Side Menu     │   Hero / Search / Summary                               │
│                 │                                                         │
│  - 概要          ├───────────────────────────┬─────────────────────────────┤
│  - 画面設計       │                           │                             │
│  - 学習コンテンツ  │   Chapter Grid            │   Quick Links               │
│  - 実装と運用     │                           │                             │
│                 ├───────────────────────────┴─────────────────────────────┤
│                 │   Featured Laws          │   Learning Path               │
│                 ├──────────────────────────┴─────────────────────────────┤
│                 │   Updates / Recommended Recipes                         │
└─────────────────┴─────────────────────────────────────────────────────────┘
```

### 含む要素

- ヒーロー: サイトタイトル・概要・検索バー
- 章グリッド: 8パートを視覚的に一覧
- クイックリンク: 人気法則・頻出レシピ
- 注目法則: ピックアップ3〜5件
- 学習パス: 初心者向け / QA向け / PdM向けの導線
- 更新情報: 最近追加されたレシピ・法則

---

## ワイヤー 2: 章トップページ

```
┌─────────────────┬─────────────────────────────────────────────────────────┐
│                 │   Breadcrumb / Chapter Title                            │
│   Side Menu     ├───────────────────────────┬─────────────────────────────┤
│  (現在の章を     │                           │                             │
│   ハイライト)    │   Chapter Overview        │   In-page TOC               │
│                 │                           │                             │
│                 ├───────────────────────────┤                             │
│                 │   Related Laws            │                             │
│                 ├───────────────────────────┴─────────────────────────────┤
│                 │   Section Cards / Learning Blocks                       │
└─────────────────┴─────────────────────────────────────────────────────────┘
```

### 含む要素

- パンくず: トップ > Part X
- 章概要: この章で学ぶこと・対象者・所要時間
- 章内目次: セクションへのアンカーリンク
- 関連法則: この章に紐づく主要法則のカード
- セクションカード: 各トピックへの入口

## ワイヤー 3: 法則詳細ページ

```
┌─────────────────┬─────────────────────────────────────────────────────────┐
│                 │   Law Title / Summary                                   │
│   Side Menu     ├──────────────────────────┬──────────────────────────────┤
│                 │                          │                              │
│                 │   Definition & Diagram   │   Use Cases / Examples       │
│                 │                          │                              │
│                 ├──────────────────────────┴──────────────────────────────┤
│                 │   QA View / Design Implications                         │
│                 ├──────────────────────────┬──────────────────────────────┤
│                 │   Bad Example (NG)       │   Good Example (OK)          │
└─────────────────┴──────────────────────────┴──────────────────────────────┘
```

### 含む要素

- 法則名・一文要約・タグ
- 定義: 原著・出典を明記
- 図解: SVG で可視化
- 適用例: 実際の UI 画面への当てはめ
- QA観点: レビュー・テスト時のチェック項目
- Before（NG）/ After（OK）の比較

## ワイヤー 4: レシピ詳細ページ

```
┌─────────────────┬─────────────────────────────────────────────────────────┐
│                 │   Recipe Title / Context                                │
│   Side Menu     ├─────────────────────────────────────────────────────────┤
│                 │   Problem → Principle → Prescription → Evaluation       │
│                 ├───────────┬───────────┬─────────────┬───────────────────┤
│                 │ Problem   │ Principle │ Prescription│ Evaluation        │
│                 ├───────────┴───────────┴─────────────┴───────────────────┤
│                 │   Before / Anti-pattern  │   After / Recommended UI    │
└─────────────────┴──────────────────────────┴────────────────────────────────┘
```

### 含む要素

- レシピ名・対象画面・関連法則タグ
- 4ステップフォーマット:
  - **Problem**: ユーザーが何に迷うか
  - **Principle**: どの法則で説明するか
  - **Prescription**: UI をどう直すか
  - **Evaluation**: 何を測るか（成功指標）
- Before: アンチパターン（問題のある UI）
- After: 推奨 UI（改善後）

---

## レシピ収録リスト（予定）

| カテゴリ | レシピ名 |
|---------|---------|
| 認証 | ログイン、パスワードリセット、オンボーディング |
| 入力 | フォーム設計、バリデーション、エラー表示 |
| 探索 | 検索、フィルタ、ソート |
| 一覧 | テーブル、カード一覧、ページネーション |
| 詳細 | 詳細表示、比較、モーダル |
| 管理 | ダッシュボード、通知、設定 |
| 状態 | 空状態、ローディング、エラー回復 |
| AI | AIチャット、Human-in-the-loop、提案UI |

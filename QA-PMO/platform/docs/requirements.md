# requirements.md — PMO統合プラットフォーム（AIなしMVP）

## 1. 誰のための何の機能か
- **対象**: 上司・見込み顧客・社内
- **目的**: PMO提供メニュー（カタログ）と、各サービスの**実working MVPツール**を1つのプラットフォームに統合して提示する
- **制約**: AI/LLMは使用しない。すべて**決定的な確立アルゴリズム/OSS**で実装する（車輪の再発明をしない）

## 2. 受け入れ基準

### プラットフォーム（共通）
| # | 要件 |
|---|---|
| P-01 | サイドナビ2階層で全サービスをナビゲートできる |
| P-02 | 品質PMO系サービスはカタログ詳細（説明・特徴・フロー・CTA）を表示 |
| P-03 | AIサービス系は「実working MVPツール」を画面内で操作できる |
| P-04 | サービス横断のインクリメンタル検索 |
| P-05 | 問い合わせモーダル |
| P-06 | バックエンド・ビルド不要（静的サイト・即デプロイ可） |

### 実working MVPツール（AIなし・確立技術）
| # | ツール | 確立技術 | 主要機能 |
|---|---|---|---|
| T-DOC | ドキュメント検証 | textlint/RedPen系ルール | 曖昧語・冗長文・必須節欠落の検出＋スコア |
| T-TRC | トレーサビリティ | RTM | 要件↔テストのカバレッジ・未カバー・孤立検出 |
| T-PLN | 計画策定 | ISO 29119-3 | フォーム入力→テスト計画書Markdown生成 |
| T-TD  | テスト設計 | 境界値/同値/ペアワイズ | テストケース自動生成＋CSV出力 |
| T-UX  | UI/UX検証 | axe-core＋ヒューリスティック | WCAG違反・代替テキスト/lang等の検出 |
| T-AUTO| テスト自動化 | Playwright/pytest/bats | UI/API/BATのテストscaffold生成 |
| T-CI  | CI/CD構築 | GitHub Actions | パイプラインYAML生成 |

### 品質基準（qa-review-standards・非交渉）
- ISTQB severity（Critical/Major/Minor/Cosmetic）で指摘を分類
- Evidence-only：検出は具体的な根拠（行番号・ルールID・要素）を伴う

## 3. 除外
- AI/LLM呼び出し（本制約）
- サーバーサイド処理・DB・認証

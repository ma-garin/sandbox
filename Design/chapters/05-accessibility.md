# Part 5: アクセシビリティ

## この章で学ぶこと

WCAG 2.2 の4原則（POUR）を理解し、設計・実装・QA の各フェーズで何をチェックすればよいかを整理する。

---

## 5-1. POUR の4原則

| 原則 | 英語 | 意味 |
|------|------|------|
| P | Perceivable（知覚可能） | 見える・聞こえる・代替できる |
| O | Operable（操作可能） | キーボードで操作できる・十分な時間がある |
| U | Understandable（理解可能） | 理解できる・予測できる・エラー訂正できる |
| R | Robust（堅牢） | 支援技術・ブラウザで正しく解釈できる |

---

## 5-2. WCAG 2.2 適合レベル

| レベル | 意味 | 義務度 |
|--------|------|--------|
| A | 最低限 | 必須 |
| AA | 標準（日本の JIS X 8341-3 はほぼ AA 準拠を要求） | 推奨（事実上必須） |
| AAA | 最高（一部コンテンツでのみ達成可能） | できる範囲で |

---

## 5-3. Perceivable（知覚可能）

### 1.1 代替テキスト

- 意味のある画像には `alt` 属性を記述する
- 装飾画像は `alt=""` を設定して読み上げをスキップ
- アイコンボタンには視覚的テキストか `aria-label` を付与

### 1.3 適応可能

- 構造は HTML の見出し（h1〜h6）で表現する（見た目だけで表現しない）
- テーブルには `<th scope>` を適切に設定する

### 1.4 判別可能

| 基準 | 内容 | レベル |
|------|------|--------|
| 1.4.1 | 色だけで情報を伝えない（アイコンや文字を併用） | A |
| 1.4.3 | テキストのコントラスト比 4.5:1 以上 | AA |
| 1.4.4 | テキストを 200% 拡大してもレイアウト崩れなし | AA |
| 1.4.11 | UI コンポーネントのコントラスト比 3:1 以上 | AA（2.2追加） |

---

## 5-4. Operable（操作可能）

### 2.1 キーボード操作

- すべての機能をキーボードのみで操作できる
- キーボードトラップが発生しない（モーダル・カルーセル等）

### 2.4 ナビゲーション

| 基準 | 内容 | レベル |
|------|------|--------|
| 2.4.1 | スキップリンクでメインコンテンツへ飛べる | A |
| 2.4.3 | フォーカス順序が論理的 | A |
| 2.4.7 | フォーカスインジケーターが視認できる | AA |
| 2.4.11 | フォーカスが隠れない（2.2追加） | AA |

### 2.5 入力方法（WCAG 2.2 追加項目）

| 基準 | 内容 | レベル |
|------|------|--------|
| 2.5.3 | ラベルにテキストを含む（UIコンポーネントのアクセシブル名） | A |
| 2.5.7 | ドラッグ操作に代替手段がある | AA |
| 2.5.8 | タップターゲットの最小サイズ 24×24px | AA |

---

## 5-5. Understandable（理解可能）

- ページの言語を `<html lang="ja">` で指定
- エラーメッセージは具体的に（「入力が無効です」ではなく「メールアドレスの形式で入力してください」）
- フォームには `<label>` を正しく紐付ける
- 予測できない変化を避ける（フォーカス時に自動で画面遷移しない）

---

## 5-6. Robust（堅牢）

- HTML を正しく構造化する（バリデーションエラーを減らす）
- WAI-ARIA を適切に使う（役割・状態・プロパティ）
- カスタムコンポーネントに適切な role と aria-* 属性を付与

### よく使う ARIA

| 属性 | 用途 | 例 |
|------|------|-----|
| `aria-label` | 要素のアクセシブル名 | `<button aria-label="閉じる">×</button>` |
| `aria-expanded` | 開閉状態 | アコーディオン・ドロップダウン |
| `aria-live` | 動的更新の通知 | 検索結果件数・トースト |
| `aria-invalid` | 入力エラー状態 | フォームバリデーション |
| `role="dialog"` | モーダルダイアログ | モーダル要素 |

---

## 5-7. QA チェックリスト

### ツール

| ツール | 用途 |
|--------|------|
| axe DevTools | 自動チェック（Chrome拡張） |
| WAVE | 視覚的なアクセシビリティ評価 |
| Lighthouse | Chrome DevTools 内の自動監査 |
| NVDA / VoiceOver | スクリーンリーダー実機確認 |
| Colour Contrast Analyser | コントラスト比の確認 |

### テスト項目

- [ ] キーボードのみで全機能を操作できるか
- [ ] フォーカスインジケーターが常に見えるか
- [ ] スクリーンリーダーで意味が伝わるか（見出し・ランドマーク・フォーム）
- [ ] コントラスト比 4.5:1 以上か（テキスト）
- [ ] タップターゲット 24px 以上か
- [ ] 画像に適切な alt があるか
- [ ] エラーメッセージが具体的か・aria-invalid と紐付いているか
- [ ] ページ言語 `lang` が設定されているか

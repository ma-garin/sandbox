# Part 7: QAエンジニア向けUX検証

## この章で学ぶこと

機能テストで終わらず、**体験品質（UX Quality）** まで検証できる QAエンジニアになるための観点・手法・成果物を整理する。

---

## 7-1. 機能テスト → UX テストへの拡張

| 従来のQA視点 | UX視点への変換 |
|-------------|--------------|
| 機能が動くか | 成功・失敗・再開がユーザーに伝わるか |
| 入力できるか | 入力エラーが理解できるか・修正できるか |
| 画面が表示されるか | 初見で次の操作が分かるか |
| エラーが出るか | エラーメッセージが具体的で解決策があるか |

---

## 7-2. QA × UX のテスト観点マップ

### 機能テスト

| QA観点 | UXへの変換 | チェック例 | 成果物 |
|--------|-----------|-----------|--------|
| 正常系 | 達成感・完了体験の品質 | 完了後のフィードバックは適切か | UX付きテストケース |
| 異常系 | エラー回復の支援 | エラーメッセージは具体的か | エラー体験チェックシート |
| 境界値 | 極端操作時の体験 | 長い文字列・特殊文字時の表示崩れ | 境界値UXメモ |

### 探索的テスト

| 視点 | 観察ポイント | 記録方法 |
|------|------------|---------|
| 迷い | 操作を止めた・戻ったポイント | セッションシート |
| 違和感 | 「あれ？」と感じた瞬間 | セッションシート |
| 不安 | 「本当に送信していいのか？」 | セッションシート |
| 誤操作 | 意図しない操作・誤クリック | セッションシート |

**セッションシートテンプレート**

```
日時:
テスト対象:
テスター:

観察ログ（時刻 / 操作 / 観察事項）:
- HH:MM | [操作] | [気づき]

サマリー（重大な問題 / 気になる点 / 良かった点）:
```

### 互換性テスト → UX品質

| 端末・環境 | UX観点チェック |
|-----------|--------------|
| スマートフォン（iOS/Android） | タップターゲットサイズ / 片手操作可能か |
| タブレット | レイアウトの崩れ / 文字サイズ |
| 折りたたみ端末 | 画面サイズ変化時の追従 |
| 低解像度ディスプレイ | コントラスト・可読性 |
| 低速ネットワーク | ローディング状態の表示 / 空白時間の体験 |

### 非機能テスト → 体験品質

| 非機能要件 | 体験品質への変換 | 閾値目安 |
|-----------|---------------|---------|
| レスポンスタイム | 操作後のフィードバック速度 | 400ms 以内（Doherty）|
| ページロード | 初回表示の待機体験 | 3秒以内（LCP）|
| 可用性 | 障害時のフォールバック体験 | 503 ページの品質 |

## 7-3. UX付きテストケースの書き方

### 従来のテストケース

```
TC-001: ログインボタンをクリックする
期待: ダッシュボードに遷移する
```

### UX 観点を加えたテストケース

```
TC-001: ログイン後の遷移とフィードバック
操作: 正しい認証情報でログインボタンをクリック
機能期待: ダッシュボードに遷移する
UX期待:
  - ボタンクリック後 400ms 以内に処理中インジケーターが表示される
  - 遷移後、「ようこそ、○○さん」等の個人化されたメッセージがある
  - 前回アクセスページや継続中タスクが分かる
UX NG例:
  - クリック後に無反応で固まって見える（フィードバックなし）
  - ダッシュボードが真っ白で「読み込み中」の表示もない
```

# Part 8: AI/エージェント時代のUX

## この章で学ぶこと

AIが出力を生成し、エージェントが自律的に動作する時代における新しいUX設計の原則と、それをQA・設計でどう扱うかを整理する。

---

## 8-1. AI時代のUXが従来と異なる点

| 従来のUI | AI搭載UI |
|---------|---------|
| 出力は決定的（同じ入力 → 同じ出力） | 出力は確率的（同じ入力でも異なる出力になりうる） |
| エラーは明確 | 「間違いか合っているか」が曖昧 |
| ユーザーが主体 | AIとユーザーが協調 |
| フローは線形 | 対話的・非線形 |

---

## 8-2. AIチャット UI の設計原則

### 必須要素

| 要素 | 説明 | 実装例 |
|------|------|--------|
| 根拠・出典 | AI 回答の信頼性を示す | 「参考: ○○」リンク |
| 再生成 | 別の回答を得られる | 「再生成」ボタン |
| 修正指示 | 回答を改善できる | 「もっと詳しく」「要約して」ボタン |
| 取り消し | 操作を戻せる | Undo / 「前の状態に戻す」 |
| 履歴 | 過去の対話を参照できる | チャット履歴サイドバー |
| コピー | 回答を利用できる | コードブロックのコピーボタン |

### AIチャットレシピ

| ステップ | 内容 |
|---------|------|
| Problem | AI の回答が正しいか不正確かユーザーが判断できない |
| Principle | システム状態の可視化（#1）/ ユーザー制御と自由（#3） |
| Prescription | 根拠・出典表示。再生成・修正ボタン。確信度のインジケーター（あれば）|
| Evaluation | 再生成率（低いほど良い）/ コピー率（高いほど有用）/ タスク完了率 |

---

## 8-3. Human-in-the-loop（HITL）のUX

AI が判断し、人間が承認・介入するフローの設計原則。

### 設計パターン

```
AI 処理
  ↓
AI の判断 + 根拠の提示
  ↓
人間によるレビュー・承認・却下
  ↓
実行 or 差し戻し
  ↓
監査ログへの記録
```

### HITL UIの必須要素

| 要素 | 説明 |
|------|------|
| AI の判断理由 | なぜその判断をしたかを平易に説明 |
| 信頼スコア / 確信度 | 高信頼なら自動実行、低信頼なら人間確認に分岐 |
| 承認 / 却下ボタン | 明確なアクション |
| 差し戻しコメント | AI への再指示 |
| 監査ログ | 誰がいつ何を承認したかの記録 |
| 説明責任 | 最終判断者を明示する |

## 8-4. Agent-friendly UI（AIエージェントが使いやすい UI）

AIエージェントが UI を操作する場合（RPA・Computer Use・Web Automation）の設計指針。

### 問題

従来のUIはエージェントにとって難しい:
- 視覚的手がかりに依存している（色だけで状態を表現）
- ラベルが曖昧（「送信」vs「確認」の区別が文脈依存）
- 動的に変わるDOM構造

### 設計ガイドライン

| 観点 | 推奨 | 避けること |
|------|------|-----------|
| ラベル | 意味が明確なラベルテキスト | アイコンのみ・色のみで表現 |
| 状態 | `aria-*` や `data-*` で状態を明示 | CSS クラスのみで状態を表現 |
| DOM構造 | 安定したセレクター（id / data属性） | 動的に変わるクラス名 |
| エラー | `aria-invalid` + エラーメッセージ | 赤色のみ |
| フォーム | `<label>` の正しい紐付け | placeholder のみ |

### テスト観点

- [ ] スクリーンリーダーで操作できるか（＝エージェントにも読める）
- [ ] 安定した id / data 属性でターゲット指定できるか
- [ ] 状態変化が DOM 属性に反映されるか

---

## 8-5. AI UX のアンチパターン

| アンチパターン | 問題 | 改善策 |
|--------------|------|--------|
| 黒箱AI | なぜその答えを出したか分からない | 根拠・参照を表示する |
| 全自動化 | 人間の確認なく重大な操作を実行 | 重大操作には HITL を挟む |
| 過信誘導 | AI の回答が常に正しいと誤認させる | 確信度表示・免責表示 |
| 取り消し不可 | AI が行った操作を元に戻せない | Undo / 下書き保存を用意する |
| 無限ローディング | AI 処理中に状態が分からない | 進捗表示・タイムアウト処理 |

---

## 8-6. QA観点：AI機能のテスト

| テスト観点 | チェック項目 |
|-----------|------------|
| 出力の多様性 | 同じ入力を繰り返し、有害・不正確な出力が出ないか |
| エラーハンドリング | API タイムアウト・エラー時のフォールバックが機能するか |
| HITL フロー | 承認・却下・差し戻しのフローが正しく動くか |
| 監査ログ | 誰がいつ何をしたかが正しく記録されるか |
| 状態の可視化 | 処理中・完了・エラーが適切に表示されるか |
| エージェント対応 | aria 属性・ラベルが正しく設定されているか（自動テストで検証可能） |

# 実バグデータセット — Saleor Commerce

> **収集方法の正直な開示**
> GitHub API はレート制限（60req/h 未認証）のため直接取得できず。
> WebFetch（公開GitHub HTML）+ WebSearch（Google）で取得。
> 動的レンダリングによりissueリストページは取得不能。
> → 個別issueページ直接アクセス＋リリースノート参照の組み合わせ。
> 収集できたのは **20件**（事前登録目標30件に対して2/3）。
> 件数不足は結果セクションで正直に記載する。

---

## 選定基準（PRE_REGISTRATION.md より再掲）

- 対象ドメイン: Checkout / Payment / Order / Inventory / Discount / Tax
- 期間: 2023年〜2024年（一部2022年末を含む ← 境界近傍のため採用）
- 除外: ドキュメント/CI/インフラのみ
- ステータス: Closed 優先（Open でも根本原因が特定されていれば採用）

---

## バグカテゴリ定義（採点で使用）

| コード | 意味 |
|---|---|
| **CONC** | 競合状態・同時実行・冪等性 |
| **STATE** | 状態管理・状態遷移ミス・孤立レコード |
| **DATA** | データ集計・計算・精度エラー |
| **VALID** | バリデーション抜け・ガード節不足 |
| **SEC** | セキュリティ（認可・暗号・認証） |
| **I18N** | 国際化・通貨・言語設定 |

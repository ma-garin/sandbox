# Harness. ToDo管理システム — 実装（デザインモックの落とし込み）

`../gallery.html` のデザインモックを、動作する静的Webアプリとして実装したもの。
ビルド不要・素の HTML/CSS/JS で構成し、GitHub Pages でそのまま公開できる。

## 起動

```bash
# そのままブラウザで開く
open index.html            # macOS
# もしくは簡易サーバ経由（file:// で不都合がある場合）
python3 -m http.server 8000   # → http://localhost:8000/
```

## 構成

```
app/
├── index.html        エントリ（#app にマウント）
├── css/styles.css    デザイントークン（ライト/ダーク）＋全コンポーネント
└── js/
    ├── icons.js      インラインSVGアイコン（currentColor 継承）
    ├── data.js       モックデータ（タスク・プロジェクト・通知など）
    └── app.js        ルーティング・状態管理・ビュー・操作
```

## 実装した画面・状態

| 区分 | 内容 |
|---|---|
| 画面 | ログイン / ホーム / ToDo一覧 / マイページ / 設定 |
| テーマ | ライト・ダーク・システム追従（トークン切替のみ、`localStorage` 永続化） |
| サイドバー | 折りたたみ（アイコンのみ表示）切替 |
| モーダル・状態 | 新規タスク作成 / 削除確認 / 通知ドロップダウン / 空状態 / ログインエラー |

## 動作する主なインタラクション

- ハッシュルーティング（`#/home` `#/todos` `#/me` `#/settings`）
- ToDo一覧: タブ切替・タスク名絞り込み・優先度の列フィルタ・表示件数/ページネーション
- タスクのチェック（完了/進行中トグル）・削除（確認ダイアログ）
- テーマ切替（トップバー or 設定＞表示）・サイドバー折りたたみ
- `(デモ)` リンクで空状態・ログインエラーをプレビュー可能

## 設計メモ

- 状態は `state` に集約し、変更のたびに `render()` で全体再描画するシンプル構成。
- データ更新は immutable（`map`/`filter` で新配列を生成し元を破壊しない）。
- 配色・角丸・影などは `:root` / `[data-theme="dark"]` のCSS変数に集約。
- モックの `gallery.html` 由来のトークン（primary `#1976D2` 等）に準拠。
- 検証: Chromium で全画面をスクリーンショット比較・JSエラーゼロを確認済み。

## 3. 非機能要件 → 検証方法

| 要件 | 検証 | 工程 |
|---|---|---|
| NFR-01 抽出率95%以上 | サンプル各形式3件で実測 | 総合 |
| NFR-02 **書式消失の検出** | 打ち消し線・強調を含むテストファイルで100%検出（TV-07） | 結合 |
| NFR-03 1,000件/30分（層2有効時60分） | 性能測定 | 総合 |
| NFR-04 メモリ2GB以内 | 実行時計測 | 総合 |
| NFR-05 macOS/Win11 | 両環境でスモークテスト | 総合 |
| NFR-06 3ステップ | 操作実測 | 受入 |
| NFR-07 理解容易性 | U-3による確認 | 受入 |
| NFR-08 継続処理 | 破損混入テスト（TV-02） | 結合 |
| NFR-09 決定性 | 層1・層3は出力ハッシュ比較。層2は差分提示を確認 | 結合 |
| **NFR-10 外部通信ゼロ** | ネットワーク監視下で実行（層2有効時も） | 総合（**必須**） |
| NFR-11 ログ非出力 | ログ全文検査（TV-54） | 結合 |
| NFR-12 認証情報非埋込 | 静的検査・レビュー | 実装 |
| NFR-13 プラグイン性 | 形式・操作の追加が新規ファイルのみで完結（TV-09） | 設計レビュー |
| NFR-14 行数・純粋関数 | 静的検査・レビュー | 実装 |
| NFR-15 カバレッジ80% | 計測（diagnosis / verify は必須） | 単体 |
| NFR-16 環境構築 | クリーン環境で確認 | 総合 |
